# コントリビューション・ガイドライン

ShoCode リポジトリへようこそ！このコミュニティ主導のイベントへの貢献に
興味を持っていただきありがとうございます。このドキュメントでは、参加登録、
リポジトリの改善、イベントそのものの形成など、貢献の方法を説明します。

## イベント参加登録

最も基本的な貢献方法は、プロフィールページを提出して**参加者として登録する**ことです。
詳しくは [README](README_jp.md) をご覧ください。簡単にまとめると:

1. **Pull Request** — リポジトリをフォークし、`profiles/your_username` 配下に
   プロフィールページを追加して、PR を提出します。
2. **@copilot に依頼** — 登録用 Issue にコメントすると、Copilot が PR を作成してくれます。

## リポジトリへのコントリビューション

イベントやリポジトリを改善するコントリビューションを歓迎します。
ツール、ドキュメント、翻訳、自動化、Issue テンプレートなど、あらゆる改善を受け付けています。

コントリビューションの提出方法:

1. リポジトリをフォークし、変更用の新しいブランチを作成する。
2. 変更を加え、明確でわかりやすい構成にする。
3. 変更内容とその理由を説明した Pull Request を提出する。

ガイドライン:

- スコープを絞る — 1 つの PR で 1 つの論理的な変更に集中する。
- 新しい機能（自動化、CI など）を追加する場合は、動作の仕組みを説明する。
- バイリンガル（英語 + 日本語）でのコントリビューションは特に歓迎します。
- レビューでのフィードバックに対して、柔軟に対応する姿勢を持つ。

## バグ報告

壊れたリンク、フォーマットの問題、わかりにくい説明など、問題を見つけた場合は、
以下の内容を含む Issue を作成してください:

- 問題の明確な説明。
- 再現手順（該当する場合）。
- **"bug"** ラベルを付ける。

## アイデア・機能リクエスト

ShoCode をより良くするアイデアがありますか？ **"enhancement"** ラベルを付けた
Issue を作成し、以下を記載してください:

- 実現したいこと。
- それがイベントやコミュニティにどう貢献するか。
- 関連する背景、モックアップ、参考事例があれば記載。

## ディスカッション

このリポジトリはコミュニティの議論の場でもあります。
Issue や Pull Request を使って会話を始めてください。
敬意を持ち、トピックに沿った建設的なやり取りを心がけましょう。
[行動規範](CODE_OF_CONDUCT_jp.md)もご確認ください。

プロフィールページの提出、タイポの修正、大胆な新しいアイデア —
どんな貢献も大切です。ShoCode に関わってくださり、ありがとうございます。
