# Squad Decisions

## Active Decisions

### Agenda & Format (Sakura)
One-day event 09:30–18:30. Keynote + 3 standard talks + 5 lightning talks in the morning. 2.5-hour hackathon after lunch (13:30–16:00). Team demos, community awards, and open networking to close. Speakers present in preferred language; slides must be bilingual. No simultaneous translation — bilingual MC summarizes between talks.
→ Full proposal: [decisions/inbox/sakura-agenda-vision.md](decisions/inbox/sakura-agenda-vision.md)

### Venue & Logistics (Naruto)
Target 50–100 attendees. Must-haves: reliable WiFi (100 Mbps+), adequate power, AV, and food service area. Fujisawa/Enoshima recommended as best balance of transit access, infrastructure, and Shonan vibe. Kamakura strong #2. Budget estimate ¥4,400–¥7,300/person. Dual-screen bilingual AV setup proposed.
→ Full proposal: [decisions/inbox/naruto-venue-logistics-vision.md](decisions/inbox/naruto-venue-logistics-vision.md)

### Speaker Program & CFP (Kakashi)
10 agentic coding topic categories. Three CFP submission paths: classic PR, GitHub Issue Form, and @copilot-assisted. Code artifact required from all speakers. 50% community slots reserved for speakers with ≤2 prior talks. Pair-speaking (JP+EN co-presenters) encouraged. Keynotes invited; community speakers reviewed blind.
→ Full proposal: [decisions/inbox/kakashi-speaker-cfp-vision.md](decisions/inbox/kakashi-speaker-cfp-vision.md)

### Branding & Sponsors (Tsunade)
Ocean-inspired palette (blues, sand white, sunset coral). "ShoCode: Agentic Coding on the Coast." Four sponsor tiers: Wave (infrastructure), Sun (experience), Shell (swag/infra), Beach (community). Non-negotiable rules: no recruiting booths, no marketing talks, no attendee data sharing. Community-first positioning — "for builders, not suits."
→ Full proposal: [decisions/inbox/tsunade-brand-community-vision.md](decisions/inbox/tsunade-brand-community-vision.md)

### Attendee Experience & Registration (Hinata)
Three registration paths: PR (classic), @copilot-assisted (comment on pinned issue), and GitHub Issue Form (structured YAML). Bilingual registration form created at `.github/ISSUE_TEMPLATE/registration.yml`. Fields include display name, role preference, language preference, dietary/accessibility needs. Automation stretch goal for auto-generating profile PRs from issues.
→ Full proposal: [decisions/inbox/hinata-attendee-experience-vision.md](decisions/inbox/hinata-attendee-experience-vision.md)

### Quality Standards (Neji)
Audited existing repo content. Five bilingual quality standards proposed: BQ-1 Parallel Content (all docs in both languages), BQ-2 Natural Language (no literal translations), BQ-3 Cross-Linking (badge links between versions), BQ-4 Template Consistency (identical fields both languages), BQ-5 Issue/PR Template Parity. Key gaps identified: CoC and CONTRIBUTING.md are English-only; profile templates inconsistent; older issue/PR templates outdated.
→ Full proposal: [decisions/inbox/neji-quality-standards-vision.md](decisions/inbox/neji-quality-standards-vision.md)
