## Three-Act Format / 三幕構成

ShoCode follows a **three-act structure** every edition:

ShoCodeは毎回 **三幕構成** で運営されます：

1. **Act 1 — Presentations / プレゼンテーション** (10:30–12:25)
2. **Act 2 — Hackathon / ハッカソン** (13:15–16:10)
3. **Act 3 — Demos & Closing / デモ＆クロージング** (16:10–18:00)

## Hackathon Structure / ハッカソン構成

### Overview / 概要

- **Build time / ビルド時間:** 2.5 hours (150 min) — 13:25–15:55
- **Team size / チームサイズ:** 1–4 people / 1〜4名
- **Team formation / チーム編成:** Pre-formed teams welcome; solo hackers join during lunch mixer / 事前結成チーム歓迎、ソロ参加者はランチ中のミキサーで合流

### Theme Prompts / テーマプロンプト

Three optional themes to spark ideas (freeform also welcome):

アイデアのきっかけとなる3つの任意テーマ（自由テーマも可）：

| # | Theme / テーマ | Description / 説明 |
|---|---|---|
| 1 | **Agent Autopilot / エージェントオートパイロット** | Build an agent that ships a feature end-to-end with minimal human input / 最小限の人間の関与でエンドツーエンドの機能をリリースするエージェントを開発 |
| 2 | **Human + Agent Pair / 人間＋エージェントペア** | Design a workflow where a human and an agent collaborate in a tight loop / 人間とエージェントが密に連携するワークフローを設計 |
| 3 | **Agent Infra / エージェントインフラ** | Build tooling, observability, or guardrails for agentic coding systems / エージェントコーディングシステム向けのツール、監視、ガードレールを構築 |

*Edition-specific themes: [TBD] / エディション固有のテーマ: [TBD]*

### Rules / ルール

1. All code must be written during the hackathon window (13:25–15:55). Pre-existing libraries/frameworks are fine; pre-built projects are not.
   全てのコードはハッカソン時間内（13:25–15:55）に書くこと。既存のライブラリ/フレームワークは可、事前構築プロジェクトは不可。
2. Must use at least one agentic coding tool or technique.
   少なくとも1つのエージェントコーディングツール/技術を利用すること。
3. Must be demo-able at the end (even if rough).
   終了時にデモ可能であること（粗削りでも可）。
4. Code must be pushed to a public GitHub repo by 15:55.
   15:55までにパブリックGitHubリポジトリにプッシュすること。

## Demo Format / デモ形式

| Parameter / パラメータ | Value / 値 |
|---|---|
| **Total demo time / デモ合計時間** | 60 min |
| **Target teams / 目標チーム数** | 8–12 teams / 8〜12チーム |
| **Time per team / チームあたりの時間** | 5 min (3 min live demo + 2 min Q&A) |
| **Changeover / 入替時間** | 1 min between teams (MC fills) / チーム間1分（MCがつなぐ） |
| **Timer / タイマー** | **Strict — MC cuts at 5 min, no exceptions / 厳格 — MCが5分で終了、例外なし** |
| **Format / 形式** | Live demo strongly preferred over slides / スライドよりライブデモ推奨 |
| **Setup / セットアップ** | Screen shared on main projector, present from own laptop / メインプロジェクターに画面共有、自分のPCで発表 |

**Adjustments / 調整:** Fewer teams → extend Q&A time. More than 12 → tighten to 4 min each.
チーム数が少なければ質疑を延長。12チーム超なら各4分に短縮。

## Community Awards / コミュニティアワード

No formal competition — community-voted recognition instead.

正式なコンペではなく、コミュニティ投票による表彰です。

| Award / アワード | Criteria / 基準 |
|---|---|
| 🏖️ **Shonan Spirit Award / 湘南スピリット賞** | Most creative or joyful hack / 最もクリエイティブまたは楽しいハック |
| 🤖 **Agent Whisperer Award / エージェントウィスパラー賞** | Best use of agentic coding / エージェントコーディングの最も優れた活用 |
| 🎤 **Best Demo Award / ベストデモ賞** | Most compelling presentation / 最も説得力のあるプレゼンテーション |

- **Voting / 投票:** All attendees vote via [TBD: Google Form / GitHub reactions] / 全参加者が投票
- **Prizes / 賞品:** Stickers, small trophies, bragging rights (no cash) / ステッカー、小トロフィー、自慢の権利（現金なし）
- **Every team that demos gets recognized** by name in the closing ceremony / デモした全チームがクロージングで名前を紹介される

## Built-in Buffers / 組み込みバッファ

~30 minutes of buffer time is distributed across breaks and transitions. The MC should keep transitions snappy to protect the hackathon build time.

約30分のバッファ時間が休憩と遷移に分散されています。MCはトランジションをスムーズに進め、ハッカソンのビルド時間を確保してください。
