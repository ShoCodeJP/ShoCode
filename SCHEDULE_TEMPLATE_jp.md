## 三幕構成 / Three-Act Format

ShoCodeは毎回 **三幕構成** で運営されます：

1. **第1部 — プレゼンテーション / Presentations** (10:30–12:25)
2. **第2部 — ハッカソン / Hackathon** (13:15–16:10)
3. **第3部 — デモ＆クロージング / Demos & Closing** (16:10–18:00)

## ハッカソン構成 / Hackathon Structure

### 概要

- **ビルド時間:** 2.5時間（150分）— 13:25–15:55
- **チームサイズ:** 1〜4名
- **チーム編成:** 事前結成チーム歓迎、ソロ参加者はランチ中のミキサーで合流

### テーマプロンプト / Theme Prompts

アイデアのきっかけとなる3つの任意テーマ（自由テーマも可）：

| # | テーマ | 説明 |
|---|---|---|
| 1 | **エージェントオートパイロット** | 最小限の人間の関与でエンドツーエンドの機能をリリースするエージェントを開発 |
| 2 | **人間＋エージェントペア** | 人間とエージェントが密に連携するワークフローを設計 |
| 3 | **エージェントインフラ** | エージェントコーディングシステム向けのツール、監視、ガードレールを構築 |

*エディション固有のテーマ: [TBD]*

### ルール

1. 全てのコードはハッカソン時間内（13:25–15:55）に書くこと。既存のライブラリ/フレームワークは可、事前構築プロジェクトは不可。
2. 少なくとも1つのエージェントコーディングツール/技術を利用すること。
3. 終了時にデモ可能であること（粗削りでも可）。
4. 15:55までにパブリックGitHubリポジトリにプッシュすること。

## デモ形式 / Demo Format

| パラメータ | 値 |
|---|---|
| **デモ合計時間** | 60分 |
| **目標チーム数** | 8〜12チーム |
| **チームあたりの時間** | 5分（3分ライブデモ＋2分質疑） |
| **入替時間** | チーム間1分（MCがつなぐ） |
| **タイマー** | **厳格 — MCが5分で終了、例外なし** |
| **形式** | スライドよりライブデモ推奨 |
| **セットアップ** | メインプロジェクターに画面共有、自分のPCで発表 |

**調整:** チーム数が少なければ質疑を延長。12チーム超なら各4分に短縮。

## テンプレートの使い方 / How to Use

1. このファイルをエディションブランチ（例: `shocode-2026-fall`）に `SCHEDULE.md` としてコピー
2. 全ての `[TBD]` をエディション固有の情報で記入
3. ハッカソンのテーマプロンプトをエディションテーマに合わせて調整
4. コミットしてチームにレビュー共有
